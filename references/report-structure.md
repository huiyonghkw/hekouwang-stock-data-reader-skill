# 报告结构规范（栏目固定模板）

每只股票的报告**结构固定、只换数据**，保证可复用、3 天一期能稳定产出。指标取自 `analysis.json`。

## 公众号 8 贴图（1080×1350，V2 米白）
1. **封面**：标题钩子 + 三数字卡（取最有反差的三个：如主力净流出 / 峰值PE / 算力毛利率）
2. **价格轨迹**：年初→峰值→现价 + 市值轨迹（折线图）
3. **资金面**：主力 N 月净流入/出 + 关键交易日（发散横条；标注"涨着出货"类反差日）
4. **龙虎榜结构**：席位类型（通道盘 vs 机构长线）
5. **利润质量**：归母净利 vs 经营现金流（分组柱）+ 合同负债
6. **利润结构**：分部收入 vs 毛利结构（两条 100% 横条，点出"小收入扛大毛利"）
7. **风险线索**：估值分位 + 公司自述风险 + 股东动作（事实罗列，不下结论）
8. **结语 + 风险提示条**：一句中立总结 + 合规免责

## 头条三件套
- `正文.md`：双标题(A/B，≤30字，疑问可悬念不行) + 开头两段直给价值 + 小标题切块 + [图N]位 + #话题# + 风险提示条
- 封面 16:9（1280×720，信息居中）
- 配图 5 张（1080 宽）：把上面 2/3/5/6/7 卡做成头条配图

## 视觉
- 默认 **V2 米白**（黏土 #c15f3c 主角 / 石板蓝 #5c6b7a 对照 / 砖红 #c0392b 流出·风险）；财经也可选 V3 Google 财经风。
- **字体统一黑体**（思源黑体 + Anthropic Sans），标题/金句都用黑体，别混衬线。
- 🚫 无卡片左侧彩色竖条；强调色只放小色点/序号/数字/SVG 填充。
- 走 `_build.py` 流水线，出图即删 HTML，交付前自检填满无留白无裁切。

## 工作流
`fetch.py <代码>` → `analyze.py <out>` → 据 analysis.json 用 build_report.py 出**初稿贴图（数字已对）** → 人/LLM 按本结构润色中立复盘文案 → `screenshot.js` 出图 → 套合规框架交付。
